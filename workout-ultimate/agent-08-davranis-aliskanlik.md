## 1. Davranış Tasarım Framework — Bengisu için Uyarlanmış

**1.1. Fogg Behavior Model (B = MAP).** Davranış üç değişkenin **çarpımı** — biri sıfırsa davranış olmaz.

| Değişken | Bengisu durumu | Tasarım kaldıracı |
|---|---|---|
| **Motivation** | Yüksek (düğün, sağlık, akademik kararlılık) | Yüksek tutmak değil, **dalgalandığında yakalamak** — fluctuation-aware |
| **Ability** | Orta-yüksek (zaman uygun, salon yakın, bilim okur) | "Tiny" — minimum versiyonu görünür kıl. 30 dk yerine "5 dk yürüyüş" var |
| **Prompt** | **Zayıf** (kendi planında ama uygulama hatırlatıcı henüz olgun değil) | **En büyük fırsat** — doğru zaman + doğru ton |

**Pratik sonuç:** Uygulamanın yatırım önceliği prompt mimarisi — Bengisu'nun motivasyon düştüğü gün (düşük uyku, geç luteal, kötü mood) **ability'yi otomatik küçülterek** ve **promptu yumuşak versiyona döndürerek** davranış akmaya devam eder. "Bugün 7-8 km yürüyüşün" yerine "Bugün 20 dk hafif yürüyüş yeterli — bedenin sana 'yavaşla' diyor." MAP üçü minimumdan üst üste durduğunda davranış olur; uygulama hangisini yükselteceğini bilmeli.

**1.2. Clear "Atomic Habits" — habit stacking + identity.** Clear'ın iki çekirdek prensibi:
- **Habit stacking:** "X'ten sonra Y" zinciri. Bengisu'nun mevcut sabitleri zaten zincir hattı taşıyor (Levotiron 06:30 → ViDrate 07:00 → yürüyüş 07:30 → duş → cilt rutini → 1. öğün 10:00). Yeni davranışı **mevcut zincire bağlamak** %30 daha tutarlı oluyor (Lally 2010).
- **Identity-based habits:** "Kilo vermek istiyorum" yerine "Bedenine değer veren biriyim". Bengisu için: "Ben kuvvet antrenmanı yapan bir kadınım", "Ben cycle'ımı dinleyen biriyim", "Ben restorasyon gününü onurlandıran biriyim". Davranış bir kez yapıldığında bu kimliği **oylar**. Uygulamanın dili kimlik destekleyici olmalı — "Yaptın!" değil, "Bu sen."

**1.3. Eyal "Hooked" — Trigger → Action → Variable Reward → Investment.** Sosyal medya manipülasyonu için tasarlanmış model, **etik kullanımda** sağlık alışkanlığı için güçlü. Bengisu için:
- **Trigger (external):** sabah 06:30 Levotiron alarm, akşam check-in nudge.
- **Trigger (internal):** "yorgun hissediyorum" → uygulama açar (cycle/uyku ne diyor?).
- **Action:** minimum sürtünme — 2 tap'te tamamlanan check-in.
- **Variable reward:** **her gün aynı kutlama olmasın**. ZEN quote, arketip kart, fizyolojik içgörü, somatic prompt, kuvvet PR alkışı — havuzdan rastgele.
- **Investment:** Bengisu sisteme veri yatırır (ölçüler, RPE, mood). Ne kadar yatırsa **kişiselleştirme** o kadar derin; çıkmak istediğinde "burası bana ait" hissi.

**Etik not:** Hooked modeli Eyal'in *Indistractable* kitabında **kötüye kullanım uyarısı** taşır. Sağlık uygulaması "günlük geri gel"i bir amaç değil, **araç** olarak kullanmalı. Hedef Bengisu'nun bedeniyle ilişkisi, uygulamayla değil. "Streak'i kaçırma" tek başına shame tetikleyici — bağlamla yumuşatılmalı.

**1.4. Wood — context cues.** Wendy Wood (USC, *Good Habits, Bad Habits*) alışkanlıkların %43'ünün **otomatik** (irade değil, bağlam) olduğunu gösterir. Üç bağlam türü:
- **Ortam:** Bengisu'nun antrenman çantası kapı yanında, ViDrate buzdolabında hazır, telefon yatakta değil masada (yapılan tasarım önerileri).
- **Zamansal:** sabit saatler (06:30, 07:30, 10:00, 17:00) cycle-aware esneklik içinde sabit kalır — biyoloji bağlamında.
- **Kişisel:** mood, cycle, uyku — uygulamanın okuduğu içsel bağlam. Bengisu'nun geç luteal'de "neden böyle hissediyorum?" sorusu cycle bağlamı görünürse cevaplanır.

**1.5. Duhigg habit loop.** Cue → Routine → Reward. Bengisu için kritik **reward tasarımı**: kg azalması yavaş ve gecikmiş ödül; **anlık ödül** lazım. ZEN quote sabahın alkışı, akşam check-in'in "bedenin teşekkür ediyor" cümlesi, kuvvet artışı grafiği — anlık dopamin. Lally 2010 (PMID 19953552): yeni alışkanlık ortalama 66 günde otomatikleşir (18-254 arası). Bengisu için **ilk 30 gün** kritik pencere — uygulamanın bu pencerede prompt'u yoğun, sonrası seyrek.

## 2. Plan vs. Gerçek Log Tasarımı — Üç Katmanlı

Bengisu'nun açık talebi: "İnsanız, engeller çıkıyor — plan vs. gerçek görmek istiyorum." Bu sadece tracking değil, **bilim ve dürüstlük**. Mükemmellik teatri yerine gerçek veri.

### 2.1. Katman A — Hareket başına quick state tap (mikroveri, set arası)

**Konum:** Antrenman ekranında her hareketin yanında, set tamamlama anında.
**UI:** 3 büyük dokunma alanı (≥48px), tek başparmakla ulaşılabilir.

```
[ ✓ Yaptım ]  [ ⚡ Değiştirdim ]  [ 🛌 Atladım ]
```

**Etkileşim:** ✓ tek tap = kayıt + sonraki harekete geç. ⚡ tap → 2 sn içinde 4 büyük buton açılır:
```
Hangi yönde?
[ Ağırlık ↓ ]  [ Set/rep ↓ ]  [ Hareket değişti ]  [ Kısa süre ]
```
🛌 tap → 4 büyük buton:
```
Bugün neden?
[ Yorgunum ]  [ Reglim ]  [ Zaman dar ]  [ Canım istemedi ]
```
"Canım istemedi" **yargısız** seçenek olarak DURMALI — bilinçli çıkar. Wood: irade düşüklüğü iradesizlik değil, **alışkanlık bağlamı yetersiz**. Bengisu bunu seçerse uygulama **fırça atmaz** — sadece kayıt eder, akşam ya da Pazar haftalık özette örüntü görünür.

**Veri yapısı:**

```typescript
interface ExerciseLogEntry {
  exerciseId: string;
  date: string;           // 'YYYY-MM-DD'
  planned: {
    sets: number;
    reps: number;
    weight: number;       // kg
    rpe?: number;
  };
  actual: {
    sets: number;         // 0 = atlandı
    reps: number[];       // her set için
    weight: number[];     // her set için
    rpe?: number[];
    duration?: number;    // dk
  };
  state: 'done' | 'modified' | 'skipped';
  modifyReason?: 'weight_down' | 'volume_down' | 'exercise_change' | 'time_short';
  skipReason?: 'fatigue' | 'period' | 'time' | 'choice';
  cycleDay?: number;
  cyclePhase?: string;
  mood?: 1 | 2 | 3 | 4 | 5;
  sleepHours?: number;    // Apple Health'ten otomatik
  timestamp: number;
}
```

**İçgörü çıktısı:** Pazar haftalık özet — "Bu hafta 4/5 antrenman tamamlandı. 1 atlandı (Çarşamba — yorgunluk; uyku 5.2 saatti, vücudun haklıydı). 2 hareket modify edildi (RDL → trap-bar deadlift, ovulasyon ACL koruması). Bu 'yarım hafta' değil — **stratejik hafta**."

### 2.2. Katman B — Akşam 30 saniyelik kibar check-in

**Konum:** Bugün dashboard'da akşam blok (🌙 Akşam) içinde + opsiyonel push notification.
**Saat:** 21:00-22:00 (Bengisu'ya sor — varsayılan 21:30).

**Çekirdek soru — "Yaptın mı?" değil, "Nasıl gitti?":**

```
Bugün bedeninle nasıl bir gün geçirdin?
○  ✨ Kendimle barıştım
○  💪 Hedefe yakın hissettim
○  🌊 Akıp gitti
○  🌫️ Zorlandım
○  🌙 Bugün dinlenmem gerekti
```

**5 seçenek hiçbiri "başarısız" demiyor.** Tylka body-neutrality + Neff self-compassion: sonuç değil **deneyim** sorulur. Seçim → 1 cümle reframe:

- ✨ "Bu hissi bedenine kaydet — yarın da geleceksin."
- 💪 "Hedef değil, **yön**. Bugün doğru yönde."
- 🌊 "Sürtünmesiz günler nadirdir. Onurlandır."
- 🌫️ "Bugün senin için **bilgi**. Yarın ne deneyeceksin?"
- 🌙 "Restorasyon antrenman kadar değerli. Kuvvet **dinlenmede** inşa edilir."

**Opsiyonel mikro-textarea:** "Bedenin sana bir şey söyledi mi? (atla)" — 0-1 satır, zorunlu değil. Boş bırakmak da veri.

**Apple Health kontekst entegrasyonu:** Eğer uyku <6h veya cycle Day 1-2 ise check-in açılışında **proaktif önek** görünür:
> "Bedenin az uyumuş bugün (5h 12dk). Sorum: Kendine nasıl yumuşak davrandın?"

Soru aynı, çerçeve farklı. Bengisu için "yargılanmıyorum" hissi.

### 2.3. Katman C — Otomatik bağlam algılama (Apple Health + cycle)

**Bengisu zaten:** Apple Health uyku/adım/aktivite, cycle log günlük, RPE girişi opsiyonel. Bu üç sinyal kombinasyonu **proaktif empati** üretir — Bengisu **istemeden** uygulama "bugün hafiflet öner" diyebilir.

**Karar matrisi:**

```typescript
function getDailyContextSuggestion(d: Date): ContextSuggestion {
  const cycle = getCycleInfo(d);
  const sleep = getAppleHealthSleep(d);
  const recentRPE = getRecentRPE(d, 3); // son 3 gün ortalama
  const consecutiveTraining = getConsecutiveTrainingDays(d);

  // Otomatik "rest öner" tetikleyicileri
  if (cycle.day === 1 || cycle.day === 2) {
    return {
      tone: 'sacred',
      message: 'Bedenin haklıydı. Day 1-2 — kanın akıyor. Bugün yumuşaklık gün.',
      action: 'mobility_suggested',
      autoProtected: true,  // streak'e zarar yok
    };
  }
  if (sleep && sleep < 360) {  // 6h
    return {
      tone: 'gentle',
      message: 'Az uyumuşsun. Bugün hafifletmek bilge bir karar olabilir.',
      action: 'volume_reduce_offer',
      autoProtected: false,  // kullanıcı seçer
    };
  }
  if (recentRPE && recentRPE >= 8.5 && consecutiveTraining >= 5) {
    return {
      tone: 'firm-kind',
      message: '5+ gün yüksek yük. Bedenin onarım istiyor. Bugün restorasyon önerisi.',
      action: 'rest_strongly_suggested',
      autoProtected: false,
    };
  }
  return null;  // her şey iyi, müdahale yok
}
```

**Mevcut "Hasta / Seyahat / Yoğun regl" protected day sistemini genişletir** — manuel seçim **+** otomatik algılama. Bengisu protected etiketi seçmediğinde bile uygulama bağlamı görür ve **istemeden empati** sunar. Bu Ajan 6'nın relatedness boşluğunu doldurur — "yalnız mücadele" yerine "yanında bir bilen var" hissi.

## 3. Streak Tasarımı — Süreklilik, Şiddet Değil

Bengisu için **streak**'in shame'e dönüşmemesi P0. Ajan 6 dört risk faktörünü işaretledi: yüksek hedef × kısa zaman × düğün baskısı × disiplinli kişilik. Streak yanlış tasarlanırsa **kompulsif egzersiz tetikleyicisi** olur.

**3.1. İsimlendirme.** "Streak" → **"Süreklilik"** veya **"Akış"**. "Akış" Mihaly Csikszentmihalyi'nin flow kavramına da gönderme — performatif değil, **deneyim odaklı**. UI dili: "27 günlük süreklilik 🌱" / "Akışın 27 gün."

**3.2. Multiple streak — birleştirilmemeli.** Mevcut "yemekler+su tamam" tek streak Bengisu için yanlış model:
- Yürüyüş süreklilik (5/7 hedef)
- Kuvvet süreklilik (4-5/7 hedef)
- Beslenme süreklilik (5 öğün)
- Su süreklilik (8 bardak)
- **Dinlenme süreklilik** (haftalık 1-2 rest gün — POZİTİF metric)
- Cycle-dinleme süreklilik (faz log + faz-uyumlu davranış)

Birinde aksaklık olduğunda **diğerleri akmaya devam**. "Tek streak çöktü her şey çöktü" psikolojisi yok. Dashboard'da 6 mini-akış görünür, her biri kendi rengiyle.

**3.3. "1 günü kaçırdın" reframe — Türkçe sacred.** Mevcut "seri büyüyor" düzeltmesi iyi — bunu derinleştir:

| Olay | Eski tasarım | Yeni tasarım |
|---|---|---|
| 1 gün kaçırdı | "Streak bozuldu, tekrar başla!" | "Bugün dinlendin. Akışın 28→0 değil, 28+1 dinlenme = 29 gün bedenle bilinçli ilişki." |
| 2 gün kaçırdı | "2 gün! Acele!" | "İki gün ara verdin. Bedenin neden böyle istedi? Yarın yumuşak başla — 20 dk yürüyüş yeterli." |
| Cycle Day 1 | Streak risk | Otomatik **protected** + "Day 1 dinlenme akışın parçası — kayıp değil." |

**3.4. Cycle-aware grace — otomatik.** Mevcut "🩸 Yoğun regl" protected etiketi var. Genişlet: **Day 1-2 default protected** (kullanıcı opt-out edebilir). Geç luteal son 2 gün **soft protected** (ağır antrenman atlanırsa "tıkla, neden?" yerine "luteal son 2 gün — Sims protokolü deload öneriyor, bilinçli karar"). Bengisu cycle log girdikçe sistem otomatik bilir.

**3.5. Comeback narrative.** Mevcut sistem var (PROGRESS.md "comeback narrative"). Güçlendir: 7+ günlük süreklilik kırılmasından sonra ilk günde:
> "Hoş geldin. 12 gün sürekliliğin sonra 7 gün ara — bedenin senin için neye ihtiyaç duyduğunu biliyordu. Bugün yumuşak başlangıç. **Hedef antrenman değil, bedeninle yeniden bağlantı.**"

Antrenman planı **otomatik küçültülür** — RPE 6 hedef, volume %50, ısınma uzun. "Önce bedenle barış, sonra performans."

**3.6. Burnout detection — proaktif.** Mevcut sistem var. Sinyaller:
- 5+ gün ardarda yüksek RPE
- Uyku <6h × 3+ gün
- Mood ≤2 × 3+ gün
- "Yedikten sonra ekstra antrenman" düşüncesi tıklandı (Ajan 6)
- Compulsive screening haftalık skoru ≥3.5

**Tetiklenirse:** dashboard'da kibar nudge (kırmızı uyarı DEĞİL — sacred ton):
> "Son haftaya bedeninle birlikte bakıyoruz. Yüksek yük, az uyku, düşük enerji bir araya geldi. Bu **istemediğin bir model değil — vücudun haritası**. Önümüzdeki 3 gün için bir **restorasyon penceresi** açmak ister misin? Antrenman düşer (mobility + yürüyüş), uyku önce gelir, beslenme baseline."

Kullanıcı seçer; uygulama dayatmaz. **Autonomy korunur** (SDT).

**3.7. Streak görünürlüğü — Bengisu'ya sor.**
- A: Sürekli görünür (Bugün dashboard'da büyük rakam)
- B: Sadece haftalık özet
- C: Hidden, "Akışım" sekmesinde isteyince bakar

**Önerim B veya C** — Bengisu kompulsif risk taşıyor, sürekli görünürlük **kontrol** ihtiyacını besliyor olabilir. Açık soru §11.

## 4. Pozitif Pekiştirme — Variable Reward, Process Praise

**4.1. Variable reward havuzu.** Eyal: aynı reward → habituation. Bengisu için reward havuzu cycle/mood/zaman bağlamına göre rastgele dağıtılır:

| Reward türü | Tetik | Örnek |
|---|---|---|
| ZEN quote | Sabah açılış | Maisie Hill / Sims / Tylka — havuzdan rastgele |
| Arketip kart | Cycle faz değişimi | "Folliküler — Genç Kız enerjin uyandı" |
| Fizyolojik içgörü | Antrenman sonrası | "Bu set'te mTOR yolu aktif — 24-48 saat onarım fazı" |
| Somatic prompt | Akşam check-in | "Bedenin bugün ne öğretti?" |
| Kuvvet artışı | PR olduğunda | "Squat +2.5 kg — 3 hafta önce yapamadığın" |
| Süreklilik milestone | 7/14/28/56 gün | Sade kutlama, **performatif değil** |
| Cycle dinleme | Protected day kullanıldı | "Bedenini dinledin — bu **gücün**." |
| Restorasyon | Rest günü | "Onarım antrenman kadar değerli" |

**Variable timing + variable type.** Bengisu hangi gün hangi reward'u alacağını bilmemeli — beyin sürpriz dopamin bekler.

**4.2. Process praise > outcome praise (Dweck).** Carol Dweck Stanford 1998: "Çok zekisin" → fixed mindset (riskten kaçınma). "Çok çalıştın" → growth mindset (challenge engagement +%30). Bengisu için:

| Outcome praise (kaçın) | Process praise (kullan) |
|---|---|
| "Harikasın!" | "Bugün cycle'ını dinledin." |
| "Mükemmel!" | "RPE'ni doğru ayarladın." |
| "1 kg verdin!" | "3 hafta tutarlı kuvvet günü." |
| "Çok güzel!" | "Yorgunken modify ettin — bu **bilge bir karar**." |

Bengisu MSc — performatif övgü onun için yapay. Süreç odaklı + spesifik övgü onun için saygı.

**4.3. Ses/görsel/haptic feedback — sade.** 
- Set tamamlama: yumuşak haptic (tap), sessiz görsel onay (✓ pulse).
- Gün tamamlama: uzun haptic, ZEN cümle.
- PR: kısa sound (opsiyonel — varsayılan kapalı), arketip kart.
- **Confetti?** Mevcut sistem 5 yemek + 8 bardak için confetti tetikliyor. **Anti-kompulsif risk taşıyor** — "bütün hedefleri vurmalıyım" hissi tetikleyebilir. Önerim: confetti'yi **haftalık** milestone'a kaydır (haftada 5/7 yürüyüş), günlük tetikleyicilerden çıkar. Daha detay §8.

**4.4. Overpraising'ten kaçın.** "İnanılmazsın!", "Sen bir efsanesin!" — Bengisu için yapay ve infantilize. Sacred feminine ton "saygıdeğer kadın → saygıdeğer kadın" — sade onay.

## 5. Anti-Kompulsif Tasarım — Bengisu için P0

Ajan 6 dört kırmızı bayrak: yüksek hedef × kısa zaman × düğün baskısı × disiplinli kişilik. Compulsive exercise + disordered eating riski klinik olarak yüksek. Tasarım kuralları:

**5.1. "Daha fazla yap" çağrıları YOK.**
- ✗ "Yarın 8 km'ye çık!"
- ✗ "Bu hafta volume +%20 dene!"
- ✗ "Bonus antrenman?"
- ✓ "Bugün planın bu — bedenin yetiyor mu?" (planın altı opsiyonel)
- ✓ "Geç luteal yaklaşıyor — bu hafta yumuşak hazırlık."

**5.2. Rest günleri kutlanmalı — gizlenmemeli.** Mevcut sistemde Pazar rest var. UI'da Pazar **görünür kutlama**:
> "🌙 Restorasyon Günü. Bugünkü hediyen: kuvvet kaslarda değil, **yataktan kalkana** kadar inşa ediliyor."

**Aktif öneriler** — boşluk değil:
- Yumurta + et baseline (1500-1700 kcal hedef, deficit yok)
- Magnezyum 350 mg
- 8h uyku hedef
- Opsiyonel 10 dk mobility / yoga
- Sosyal: kız kardeşle çay, anne ile yürüyüş — relatedness inşası
- Kuvvet öğrenme: 1 podcast / Schoenfeld makalesi (pedagog Bengisu için L3 reward)

**5.3. "Bugün dinlendim" 5/7 yürüyüş süreklilik akışını BOZMAZ.**

```typescript
function isStreakProtected(d: Date, log: DailyLog): boolean {
  if (log.protectedTag) return true;  // manuel
  if (log.dailyContext?.autoProtected) return true;  // cycle Day 1-2
  if (log.actions.includes('rest_intentional')) return true;  // bilinçli rest
  if (log.skipReason === 'illness' || log.skipReason === 'travel') return true;
  return false;
}
```

Bengisu Pazar'a "rest_intentional" tıklayabilir. Süreklilik **artar** ("28 gün bedenimle bilinçli ilişki" — antrenman + rest birlikte).

**5.4. "Daha az ye" hiçbir yerde YOK.** Macro hedefler **alt sınır** olarak çerçevelenir:
- Protein 160g — **minimum**, fazlası iyi.
- Yağ 90g — **minimum**, fazlası iyi.
- Kalori 1750 kcal — **alt sınır**, üstü esnek.

**5.5. Self-compassion entegrasyonu (Neff).** Akşam check-in 🌫️ "Zorlandım" seçilirse:
> "Zor günler vardır. Bu seni başarısız yapmıyor, **insan yapıyor**. Yarın yeni gün. Şimdi bir nefes — 4-7-8. Sonra yatağa."

Neff 2013 RCT (n=51): self-compassion müdahalesi 8 hafta → eating behavior +%18, exercise consistency +%24. **"Sert ol kendine, motive ol" tersine motivasyonu çökertiyor**. Bengisu MSc — bilim sevgisi self-compassion'ı kabul etmesini kolaylaştırır.

**5.6. Compulsive screening — haftalık 3 madde (Ajan 6 önerisi).** Pazar akşam check-in'in son sorusu opsiyonel:

```
Bu hafta 1-5 arası işaretle (atlamak istersen "atla"):
- Antrenman atlamayı düşündüğümde huzursuzluk ____
- Yedikten sonra ekstra antrenman düşüncesi ____
```

Ortalama ≥3.5 → bir sonraki Pazartesi:

Ortalama ≥4 × 2 hafta üst üste → kibar uzman önerisi:

### 9.4. `getCompulsiveExerciseFlags(history)` — risk algılama

```typescript
interface CompulsiveFlags {
  level: 'green' | 'yellow' | 'orange' | 'red';
  triggers: string[];
  suggestion: string | null;
}

function getCompulsiveExerciseFlags(history: WorkoutLogEntry[]): CompulsiveFlags {
  const recent7 = history.slice(-7);
  const recent14 = history.slice(-14);
  const triggers: string[] = [];

  // 1. 5+ gün ardarda yüksek RPE
  const consecutiveHighRPE = countConsecutive(recent7, e => (e.daySummary.averageRPE ?? 0) >= 8);
  if (consecutiveHighRPE >= 5) triggers.push('5+ gün yüksek RPE');

  // 2. Uyku <6h × 3+ gün
  const lowSleep = recent7.filter(e => (e.context.sleepHours ?? 8) < 6).length;
  if (lowSleep >= 3) triggers.push('3+ gün <6h uyku');

  // 3. Mood ≤2 × 3+ gün
  const lowMood = recent7.filter(e => (e.daySummary.overallMood ?? 3) <= 2).length;
  if (lowMood >= 3) triggers.push('3+ gün düşük mood');

  // 4. Rest günü hiç yok — 14 günde
  const restDays = recent14.filter(e => e.dayPlan === 'rest' || e.daySummary.state === 'rest_intentional').length;
  if (restDays === 0) triggers.push('14 günde rest yok');

  // 5. Compulsive screening haftalık ≥3.5
  const screening = getLatestCompulsiveScreening();
  if (screening && screening.average >= 3.5) triggers.push('compulsive screening yüksek');

  // Skor
  const level: CompulsiveFlags['level'] =
    triggers.length === 0 ? 'green' :
    triggers.length === 1 ? 'yellow' :
    triggers.length === 2 ? 'orange' : 'red';

  let suggestion: string | null = null;
  if (level === 'orange') {
    suggestion = 'Son haftaya bedeninle birlikte bakıyoruz. 3 gün esnek hafta teklifi var — açar mısın?';
  } else if (level === 'red') {
    suggestion = 'Bedenin uzun süredir yüksek yük + az onarım sinyali veriyor. Önümüzdeki hafta için **restorasyon penceresi** açmak ister misin? Antrenman opsiyonel, beslenme baseline, uyku önce.';
  }

  return { level, triggers, suggestion };
}
```
